# corpus/ 语料资产目录

本目录用于落地 `docs/corpus-ingestion-spec.md`，承接 `gongwen-assistant` 的公开语料采集、清洗、审核、分层与沉淀。

核心原则只有一句话：**先固化原件，再清洗；先保来源，再谈复用；先审核分层，再进入可召回资产。**

## 目录结构

```text
corpus/
  raw/                  # 原件层：抓取原文、原始 HTML/PDF/DOC 文本、附件原件
  clean/                # 清洗层：去噪、结构归一化、脱敏后的可检索文本
  rules/                # 规则层：文种规范、审校规则、禁错清单
  excerpts/             # 片段层：优质片段、反例片段、按文种拆分的摘录
  manifests/            # 清单层：批次清单、入库批处理描述、字段模板与 schema
  metadata/             # 元数据层：索引、白名单、黑名单、标签字典
  reviewed/             # 审核层：审核记录、降级/下线记录、待审队列
  staging/              # 暂存加工区：自动化处理中间产物，不视为正式资产
  pending/              # 待核池：来源待补证、风险待确认、质量待复核材料
  templates/            # 入库与审核模板
  tools/                # 语料工具脚本、处理约定与运行说明
```

## 分层约束

### 1. raw/
- 只放原始抓取件或原始导出件。
- 禁止覆盖原件；如需重抓，新增版本或批次目录。
- 每份原件必须能追溯：来源 URL、抓取时间、抓取方式、发布主体。

### 2. clean/
- 只放从原件派生的清洗件。
- 必须可回溯到 `raw/` 中的原件或 `corpus_id`。
- 允许做去噪、结构归一化、脱敏；不允许改写原意。

### 3. rules/
- 只沉淀可回溯依据的规则，不收“感觉上的经验”。
- 规则项必须标明来源依据、适用文种、版本号、变更原因。

### 4. excerpts/
- 只收经过人工复核的高价值片段或反例片段。
- 必须绑定来源原文 ID、适用章节、适用文种、是否允许改写。
- 片段不是整篇模板，不得跨场景硬套。

### 5. reviewed/
- 审核记录、降级/下线记录与待审任务分开保存。
- 任一语料从 `待核` 到 `可用`，至少要有一条审核痕迹。

## 推荐入库流程

1. 候选发现：确认来源是否在白名单范围内。
2. 原件固化：先存 `raw/`，不得跳过。
3. 初筛：来源、完整性、风险、文种相关性。
4. 清洗处理：输出 `clean/`。
5. 元数据登记：写入 `metadata/` 或批次 `manifests/`。
6. 分层沉淀：按内容性质进入 `rules/` 或 `excerpts/`。
7. 人工复核：写审核单到 `reviewed/`。
8. 正式放行：状态更新为 `可用` / `限用`。

## 命名建议

- 批次：`YYYYMMDD-source-slug-batchNN`
- 语料 ID：`gw-{doc_type}-{date}-{seq}`
- 规则文件：`{doc_type}-{topic}-v{major.minor}.md`
- 片段文件：`{doc_type}-{section}-{slug}.md`
- 审核记录：`{date}-{corpus_id}-review.md`

## 状态约定

- `pending`：待核
- `active`：可用
- `limited`：限用
- `offline`：下线
- `rejected`：拒绝入库

## 首期落地要求

首期以公开可信语料为主，优先覆盖：
- 通知
- 请示
- 报告
- 函
- 会议纪要
- 工作方案
- 总结/汇报材料
- 实施意见/办法/细则

暂不把来源不明范文、AI 回灌稿、自媒体拼装稿放入核心层。
